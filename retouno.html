¿Por que evitar los callbacks en llamadas asincronas?

Un callback no es nada más que una función que pasa a otra función como argumento. Esta función es luego invocada dentro de la función exterior cuando se termina una rutina o acción. Es por esto último que los callbacks se usan principalmente en llamadas asíncronas.

Sin embargo usar callbacks sin un tener un buen diseño nos puede traer problemas de código dificil de leer y mantener, lo que se conoce como callback hell

Ejemplo de Callback

Por ejemplo, podemos usar un callback para obtener data via la API XMLHttpRequest

let urlData = "https://jsonplaceholder.typicode.com/todos" let userData = null btn.addEventListener("click",loadUserData) function loadUserData() { let xhr = new XMLHttpRequest(); xhr.open('GET',urlData,true) xhr.onload = function() { userData = JSON.parse(this.responseText) } xhr.send(); }

¿Que es una promesa?

Una promesa es un objeto JavaScript que representa el eventual estado de completo (o fallo) de una operación asíncrona. Nos permite trabajar con código sin que tengamos el valor final de la operación: En vez de recibir un valor, recibimos una promesa de que vamos a recibirlo en el futuro.

Las promesas nos permiten encadenar operaciones asíncronas mediante múltiples métodos .then() y manejar errores con el método .catch(), a diferencia de los callbacks

Ejemplo Promise

let promise = new Promise (resolve, reject) => { setTimeout(() => resolve("done"), 1000); } promise.then( result => console.log(result), error => console.log(error) )

¿Qué es un callback hell?

El callback hell es código asíncrono de JavaScript que hace uso inadecuado de los callbacks, lo cual resulta en un código con una forma de pirámide "particular"

Ejemplo Callback Hell

let form = document.getElementById("form") form.addEventListener("submit", e => { let name = document.getElementById("name").value if (name === undefined) { } })
